# STAMB-State v1.1

v1.1 extends the frozen v1 benchmark without overwriting it. It keeps the same latest-valid-state retrieval task and evaluator-only gold contract, then adds harder coverage for plan-not-done, stale mentions, time-role conflicts, scope/facet collisions, and answerability.

- events: 106
- cases: 72
- scopes: 16
- answerability: {'answerable': 60, 'insufficient_evidence': 6, 'unknown_current': 6}
- operations: {'next_action': 8, 'state_lookup': 49, 'state_summary': 15}

## Split Scopes

- train: aaai_memory, amp_project, sql_lab_q6, robot_nav, thesis_ch2, mobile_auth, search_index_rollout, billing_migration, eval_harness
- dev: grant_app, labeling_guideline, cache_refactor
- test: recsys_ablation, deployment_incident, ui_accessibility, api_rate_limit

## Added Coverage

- latest valid state vs latest mention
- planned action without completion record
- stale/invalidation/correction chains
- occurred_at / mentioned_at / updated_at / planned_for / deadline_at conflicts
- multi-facet summaries where only some facets changed
- answerability cases for unknown current state and insufficient evidence
- benchmark-bias guard cases around metric semantics

## Files

- `events_raw.json`: visible event stream without evaluator-only validity fields.
- `event_annotations.json`: evaluator/oracle-only state relevance, status, correction, and supersession annotations.
- `claim_annotations.json`: reserved for future claim/facet annotations.
- `cases.json`: evaluator-only query cases with gold state slots and support events.
- `splits.json`: scope-level train/dev/test split.
- `public/`: no-gold public input generated by this builder.
